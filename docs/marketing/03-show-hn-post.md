Show HN: OpenRoster — Agent bundles that run like a workspace, not a tool picker

**Body:**

Hey HN,

We built OpenRoster (https://openroster.ai) — a managed-agent marketplace where the product is agent *bundles*, not individual tools.

**The problem:** Agent marketplaces sell you one capability at a time. You still have to orchestrate the sequence, stitch outputs, and manage handoffs yourself. That's not automation — it's project management.

**What we built:** You compose bundles of agents that share a single operating loop. Example: a "Morning Ops Bundle" with inbox triage + calendar guard + daily brief. One launch, one workspace, one job done.

**How it works:**

- Browse 50+ agents (free)
- Preview any agent via sandbox chat (free) — real conversation, not a feature list
- Compose a bundle, connect Telegram, launch a managed run
- Agents are free to collect; you pay only for managed runtime (1 credit = 1 launch)

**Tech details:**

- Runs execute in sandboxed environments via Daytona
- Three runtime modes: bounded session, wake-on-Telegram, persistent workspace
- Every agent has a risk profile (Low/Medium/High) with specific capability disclosures — no blanket permission grants
- Failed launches are refunded automatically

**What's different:**

- Bundle-first: the mental model is compositions, not individual capabilities
- Preview-first: understand tone and logic before any money changes hands
- Risk is a feature, not fine print: plain-English labels, not compliance legalese

We're live. No waitlist, no invite codes. Would appreciate any feedback on the catalog, pricing model, or bundle UX.
