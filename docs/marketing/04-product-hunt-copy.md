## Full Description

### The problem

Agent marketplaces sell you one capability at a time. An inbox sorter here, a calendar bot there. Then you stitch them together yourself — remembering the sequence, managing handoffs, wiring outputs to inputs.

That's not automation. That's overhead with a better UI.

### What OpenRoster does differently

OpenRoster is built around *bundles* — groups of agents that share a single operating loop.

**Morning Ops Bundle:** Inbox triage + calendar guard + daily brief. One launch, one workspace, done before standup.

**Founder Inbox Bundle:** Priority sorter + reply drafter + follow-up tracker. Controlled decision queue out of message chaos.

**Weekly Review Bundle:** Notes digester + decision summary + next-week planner. Close the loop in one pass.

### How it works

1. **Browse** — 50+ agents across inbox, calendar, docs, and team workflows. All free to explore.
2. **Preview** — Chat with any agent in a sandbox. Understand its tone, logic, and boundaries before paying.
3. **Compose** — Add agents to a bundle. See the combined risk profile.
4. **Launch** — Connect Telegram, pick a runtime mode, and run. Logs, status, and artifacts in one place.

### Pricing model

- Agents are free to collect
- Preview chat is free
- You pay only for managed runtime: 1 credit = 1 successful launch
- Failed starts are refunded automatically

### Why bundles win

A single agent is a skill point. A bundle is a runnable workstation. The product gets more valuable when you think in combinations — that's where repeatable workflows and real operating leverage live.
