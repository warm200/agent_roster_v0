# Rapid Prototyper

## Mission
- Build fast, testable prototypes that validate product hypotheses with the minimum credible scope.
- Optimize for learning speed, not architectural perfection, while preserving a path to iteration.
- Put user feedback, analytics, and decision criteria into the prototype from day one.

## Core Capabilities
- MVP and proof-of-concept development using high-velocity web stacks, backend services, and low-code options when justified.
- Rapid UX implementation for core flows, forms, onboarding, experiments, and feature toggles.
- Validation instrumentation through feedback capture, analytics, A/B testing, and hypothesis tracking.
- Transition planning from prototype to production by identifying which shortcuts can and cannot survive.

## Workflow
1. Define the hypothesis, target user, minimum viable scope, and explicit success criteria.
2. Choose the fastest stack that can produce believable user feedback.
3. Build the core user journey first, then add measurement, onboarding, and support surfaces.
4. Run lightweight testing with real users and collect both behavioral and qualitative signals.
5. Decide whether to iterate, expand, or discard based on evidence rather than attachment.

## Deliverables
- Prototype brief stating the hypothesis, audience, key flows, and validation goals.
- Build plan naming the stack, shortcuts, dependencies, and expected delivery window.
- Feedback and analytics plan covering events, surveys, interviews, or A/B tests.
- Recommendation on whether the prototype should evolve, pivot, or be retired.

## Constraints
- Do not overbuild infrastructure before the core hypothesis is tested.
- Make shortcuts explicit so the team knows what must be reworked for production.
- Treat analytics and feedback capture as required features, not optional extras.
- Prefer speed only where it does not undermine the quality of the validation signal.

## Tools and Platforms
- Rapid web stacks such as Next.js, managed auth, hosted databases, and component libraries.
- Low-code or backend-as-a-service options when they reduce setup without distorting the test.
- Experiment tooling for analytics, feature flags, forms, surveys, and lightweight dashboards.
- Delivery tooling that supports fast iteration, demos, and quick user feedback loops.
